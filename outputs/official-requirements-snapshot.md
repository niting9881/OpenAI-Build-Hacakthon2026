# Official OpenAI Build Week Requirements Snapshot

Verified July 16, 2026 against the [hackathon overview](https://openai.devpost.com/) and [official rules](https://openai.devpost.com/rules). Recheck immediately before submitting because organizer updates control.

## Deadline

July 21, 2026 at 5:00 PM Pacific Time.

## Required project and submission material

- A working project built with Codex and GPT-5.6.
- One category: Apps for Your Life, Work & Productivity, Developer Tools, or Education.
- A project description explaining features and functionality.
- A public YouTube demonstration video under three minutes with audio covering the working project and how both Codex and GPT-5.6 were used.
- A repository URL. It may be public with relevant licensing, or private and shared with `testing@devpost.com` and `build-week-event@openai.com`.
- README setup instructions, sample data where needed, testing guidance, Codex collaboration details, key human decisions, and GPT-5.6/Codex contribution.
- The `/feedback` Codex Session ID for the project thread where most core functionality was built.
- A working project link, functioning demo, or test build available free to judges through the judging period.
- English submission material.

## Recommended category

Developer Tools, because OpsPilot is an agentic DevOps/security workflow. Work & Productivity is a credible alternative, but only one category should be selected.

## Equally weighted judging criteria

- Technological Implementation
- Design
- Potential Impact
- Quality of the Idea

## Important compliance checks

- Confirm entrant/team eligibility directly from the official rules.
- Use only authorized third-party SDKs, APIs, fixtures, and media.
- Do not include unlicensed music or third-party marks in the video.
- Make the project behave as depicted in the submitted video and description.
- Keep the judge-accessible build available through the announced judging period.
